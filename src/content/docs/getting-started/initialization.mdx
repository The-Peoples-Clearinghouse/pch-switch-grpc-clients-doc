El cliente gRPC requiere tres parámetros principales para establecer la conexión: el identificador del participante (`fspId`), la dirección del servidor, y los archivos de certificados.
Una vez inicializado, el cliente establece el stream autenticado mediante `StartStream` antes de que puedas enviar cualquier operación.

## Inicializar y conectar

<Tabs syncKey="lenguaje">
  <TabItem label="TypeScript">
    ```typescript
    import {
      GrpcSyncApiClient,
      IGrpcClientSecurityOptions,
      CryptoAndCertHelper,
    } from "@pch-private/interop-apis-bc-grpc-sync-client-lib";
    import { readFileSync } from "fs";

    const clientSecurityOptions: IGrpcClientSecurityOptions = {
      mode: "SERVER_TLS",
      caCertFilePath: "./certs/hub-intermediate.pem",
      clientPrivateKeyFilePath: "./certs/participante.key",
      clientCertFilePath: "./certs/participante.pem",
    };

    // Lee el certificado del participante como texto PEM
    const clientCertPem = readFileSync(
      clientSecurityOptions.clientCertFilePath!,
      "utf-8"
    );

    // CryptoAndCertHelper maneja la firma y validación criptográfica del flujo StartStream
    const cryptoAndCertHelper = new CryptoAndCertHelper(
      clientSecurityOptions.clientPrivateKeyFilePath!,
      clientSecurityOptions.caCertFilePath!
    );

    const grpcClient = new GrpcSyncApiClient(
      "switch.camara.mx:443",    // dirección del servidor gRPC
      "MIBANCO",                 // fspId: debe coincidir con el del certificado
      clientCertPem,
      cryptoAndCertHelper,
      clientSecurityOptions
    );

    await grpcClient.init();
    console.log("Cliente conectado y autenticado");
    ```

<Aside type="note">
  El participante es responsable de leer y proporcionar el contenido de los archivos de certificados y llave privada.
</Aside>

<Aside type="note">
  El método `init()` / `Init()` ejecuta internamente el flujo de autenticación `StartStream` — incluyendo el reto criptográfico.
</Aside>

## Parámetros de configuración

| Parámetro | Descripción |
|---|---|
| `fspId` | Identificador del participante. Debe coincidir exactamente con el del certificado. |
| `host` | Dirección y puerto del Switch (proporcionados por la Cámara). |
| `privateKeyPath` | Ruta al archivo `participante.key`. |
| `certificatePath` | Ruta al archivo `participante.pem`. |
| `caCertificatePath` | Ruta al archivo `hub-intermediate.pem`. |

## CryptoAndCertHelper

`CryptoAndCertHelper` es la implementación de referencia proporcionada por la Cámara para el manejo criptográfico del flujo `StartStream`. Se encarga de firmar el reto enviado por el Switch y de validar la firma de respuesta del hub.

El cliente acepta cualquier objeto que implemente la interfaz `ICryptoAndCertHelper`:

```typescript
export interface ICryptoAndCertHelper {
  signString(stringToSign: string): Promise<string>;
  validateSignature(
    signedString: string,
    signature: string,
    pubKeyFingerprint: string
  ): Promise<boolean>;
}
```

La implementación de la Cámara cubre los dos métodos requeridos:

- **`signString`** — Firma el reto con la llave privada del participante usando RSA + SHA-256. Devuelve la firma en Base64.
- **`validateSignature`** — Verifica la firma del hub usando la llave pública extraída del certificado intermedio de la CA (`hub-intermediate.pem`). Antes de verificar, compara el fingerprint de la llave pública recibida contra el del certificado cargado; si no coinciden, rechaza la validación sin intentar verificar. La firma Base64 se decodifica como `binary` antes de pasarla al verificador.

<Aside type="tip">
  Puedes proporcionar tu propia implementación de `ICryptoAndCertHelper` si necesitas integrar con un HSM, un servicio de firma externo, o un algoritmo diferente. Solo asegúrate de que `signString` produzca una firma compatible con el algoritmo configurado en el Switch para tu participante.
</Aside>

## Manejar desconexiones

El cliente mantiene el stream activo, pero pueden ocurrir desconexiones por inactividad o errores de red. Registra un handler para reconectarte automáticamente.
