La Cámara proporciona clientes gRPC listos para usar. No es necesario implementar la lógica de comunicación desde cero — solo instala el paquete correspondiente a tu lenguaje y configura las credenciales.

## Requisitos previos

Antes de instalar el cliente, asegúrate de contar con:

- Los tres archivos de certificados:
  - `participante.key` — clave privada del participante
  - `participante.pem` — certificado del participante
  - `hub-intermediate.pem` — certificado de la CA Intermedia

  Consulta la sección de [Seguridad](/pch-switch-grpc-clients-doc/getting-started/security) para más detalles sobre cómo obtenerlos.

- Los paquetes de cliente gRPC para tu lenguaje, proporcionados por la Cámara:
  - **TypeScript** — disponibles en el Artifact Registry de PCH (ver instrucciones más abajo).
  - **Java** — archivo `.jar` entregado directamente por la Cámara.
  - **C#** — archivos `.nupkg` entregados directamente por la Cámara.

## Instalación

<Tabs syncKey="lenguaje">
  <TabItem label="TypeScript">
    **Requisitos**

    - Node.js 20.0 o superior
    - npm, pnpm o yarn

    **Paso 1 — Configurar el registro privado**

    Agrega las siguientes líneas al archivo `.npmrc` en la raíz de tu proyecto (créalo si no existe):

    ```ini
    @pch-private:registry=https://us-central1-npm.pkg.dev/pch-main/pch-npm-private/
    //us-central1-npm.pkg.dev/pch-main/pch-npm-private/:always-auth=true
    ```

    Esto indica a npm que los paquetes con el scope `@pch-private` deben descargarse desde el Artifact Registry de PCH.

    **Paso 2 — Autenticarse con Google Artifact Registry**

    ```bash
    npx google-artifactregistry-auth
    ```

    Este comando usa tus credenciales de Google Cloud CLI para generar un token de acceso temporal al registro privado.

    **Paso 3 — Instalar los paquetes**

    ```bash
    npm install @pch-private/interop-apis-bc-grpc-sync-client-lib
    npm install @pch-private/interop-apis-bc-grpc-sync-public-lib
    ```

    - `interop-apis-bc-grpc-sync-client-lib` — el cliente gRPC con todos los métodos para operar con el Switch.
    - `interop-apis-bc-grpc-sync-public-lib` — los tipos e interfaces públicas compartidas.
  </TabItem>
  <TabItem label="Java">
    **Requisitos**

    - Java 8 o superior
    - Maven 3.8+ o Gradle
    - El archivo JAR proporcionado por la Cámara (`grpc-client-<versión>.jar`)

    **Paso 1 — Instalar el JAR en el repositorio local de Maven**

    La Cámara te entregará directamente el archivo `.jar`. Instálalo en tu repositorio local con el siguiente comando (reemplaza `<versión>` por la indicada):

    ```bash
    mvn install:install-file \
      -Dfile=grpc-client-<versión>.jar \
      -DgroupId=com.pch \
      -DartifactId=grpc-client \
      -Dversion=<versión> \
      -Dpackaging=jar
    ```

    **Paso 2 — Agregar la dependencia a tu proyecto**

    Una vez instalado, agrégalo como dependencia normal:

    **Maven**
    ```xml
    <dependency>
      <groupId>com.pch</groupId>
      <artifactId>grpc-client</artifactId>
      <version><versión></version>
    </dependency>
    ```

    **Gradle**
    ```groovy
    dependencies {
        implementation "com.pch:grpc-client:<versión>"
    }
    ```
  </TabItem>
  <TabItem label="C#">
    **Requisitos**

    - .NET 8 o superior
    - Los archivos `.nupkg` proporcionados por la Cámara

    La Cámara te entregará directamente los archivos `.nupkg`. Colócalos en una carpeta local (por ejemplo `artifacts/nuget`) y configura NuGet para que los tome desde ahí.

    **Paso 1 — Agregar la fuente local a NuGet**

    Apunta NuGet a la carpeta donde colocaste los paquetes:

    <Tabs syncKey="os">
      <TabItem label="PowerShell">
        ```powershell
        dotnet nuget add source .\artifacts\nuget --name local-artifacts
        ```
      </TabItem>
      <TabItem label="Bash">
        ```bash
        dotnet nuget add source ./artifacts/nuget --name local-artifacts
        ```
      </TabItem>
    </Tabs>

    Alternativamente, agrega la fuente en un archivo `NuGet.config` en la raíz del proyecto:

    ```xml
    <?xml version="1.0" encoding="utf-8"?>
    <configuration>
      <packageSources>
        <clear />
        <add key="local-artifacts" value="artifacts/nuget" />
        <add key="nuget.org" value="https://api.nuget.org/v3/index.json" protocolVersion="3" />
      </packageSources>
    </configuration>
    ```

    **Paso 2 — Referenciar los paquetes en tu proyecto**

    - `Pch.GrpcSyncApi.CSharp.Client` — el cliente gRPC. **Requerido.**
    - `Pch.GrpcSyncApi.CSharp.Client.Crypto` — helper criptográfico para el flujo de autenticación. **Opcional** — omítelo si tu equipo ya cuenta con una implementación propia de `ICryptoAndCertHelper`.

    ```bash
    dotnet add <TU_PROYECTO>.csproj package Pch.GrpcSyncApi.CSharp.Client --version 1.0.3
    dotnet add <TU_PROYECTO>.csproj package Pch.GrpcSyncApi.CSharp.Client.Crypto --version 1.0.3
    ```

    O directamente en el `.csproj`:

    ```xml
    <ItemGroup>
      <PackageReference Include="Pch.GrpcSyncApi.CSharp.Client" Version="1.0.3" />
      <!-- Opcional: omite si inyectas tu propia implementación de ICryptoAndCertHelper -->
      <PackageReference Include="Pch.GrpcSyncApi.CSharp.Client.Crypto" Version="1.0.3" />
    </ItemGroup>
    ```
  </TabItem>
