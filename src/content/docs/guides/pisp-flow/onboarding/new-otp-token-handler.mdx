Si el Cliente no recibe el OTP enviado durante el paso anterior (por ejemplo, si el SMS no llega), la app PISP puede solicitar la generación de uno nuevo. El módulo 3ppi del Switch envía a tu banco una solicitud `NewOtpToken` para que generes y envíes un nuevo código al Cliente.

Para recibir estas solicitudes, debes registrar un **handler** en el cliente gRPC. El cliente invocará esta función cada vez que el módulo 3ppi solicite un nuevo OTP para un onboarding en curso.

<Aside type="note">
  Tu banco es responsable de:
  - Invalidar el OTP anterior asociado al `onboardRequestId`.
  - Generar un nuevo OTP y enviarlo por SMS al número del Cliente (`partyId`).
  - Actualizar la relación almacenada entre el `onboardRequestId` y el nuevo OTP.
  - Reiniciar el tiempo de vida del OTP. Se recomienda un máximo de **10 minutos**.
</Aside>

<Aside type="caution">
  El handler debe responder siempre dentro del tiempo límite. Si el `onboardRequestId` no corresponde a ningún onboarding en curso, responde con un `ErrorResponse` en lugar de no responder.
</Aside>

## Registrar el handler

<Tabs syncKey="lenguaje">
  <TabItem label="TypeScript">
    ```typescript
    async function newOtpTokenHandler(
      request: ServerNewOtpTokenRequest,
    ): Promise<ServerNewOtpTokenResponse> {
      const partyId          = request.getPartyid();
      const sourceFspId      = request.getSourcefspid();
      const destinationFspId = request.getDestinationfspid();
      const onboardRequestId = request.getOnboardrequestid();

      const response = new ServerNewOtpTokenResponse();
      // La fuente de la respuesta es el banco receptor (el nuestro)
      response.setSourcefspid(destinationFspId);
      // El destino de la respuesta es quien originó la solicitud
      response.setDestinationfspid(sourceFspId);
      response.setPartyid(partyId);
      response.setOnboardrequestid(onboardRequestId);

      // Verifica que el onboardRequestId corresponda a un onboarding en curso
      if (!onboardingEnCurso(onboardRequestId)) {
        const error = new ErrorResponse();
        error.setCode('ONBOARD_REQUEST_NOT_FOUND');
        error.setMessage('No se encontró un onboarding en curso para el identificador proporcionado.');
        response.setError(error);
        return response;
      }

      // Invalida el OTP anterior, genera uno nuevo y envíalo por SMS al cliente.
      // Reinicia el tiempo de vida del OTP (se recomienda máximo 10 minutos).
      await generarYEnviarNuevoOtp(partyId, onboardRequestId);

      return response;
    }

    // Registrar el handler en el cliente gRPC
    grpcClient.setNewOtpTokenRequestHandlerFn(newOtpTokenHandler);
    ```

            SourceFspId      = destinationFspId,
            // El destino de la respuesta es quien originó la solicitud
            DestinationFspId = sourceFspId,
            PartyId          = partyId,
            OnboardRequestId = onboardRequestId,
        };

        // Verifica que el onboardRequestId corresponda a un onboarding en curso
        if (!OnboardingEnCurso(onboardRequestId))
        {
            response.Error = new ErrorResponse
            {
                Code    = "ONBOARD_REQUEST_NOT_FOUND",
                Message = "No se encontró un onboarding en curso para el identificador proporcionado.",

        }

        // Invalida el OTP anterior, genera uno nuevo y envíalo por SMS al cliente.

        GenerarYEnviarNuevoOtp(partyId, onboardRequestId);

    _client.setNewOtpTokenRequestHandler(newOtpTokenHandler);

## Campos del mensaje entrante (`ServerNewOtpTokenRequest`)

| Campo | Tipo | Descripción |
|---|---|---|
| `partyId` | `string` | Identificador del Cliente (ej. número de celular). |
| `sourceFspId` | `string` | FSP que origina la solicitud (módulo 3ppi del Switch). |
| `destinationFspId` | `string` | FSP al que va dirigida la solicitud (tu banco). |
| `onboardRequestId` | `string` | Identificador del onboarding en curso para el que se solicita el nuevo OTP. |

Ver referencia completa: [ServerNewOtpTokenRequest](/pch-switch-grpc-clients-doc/reference/server-new-otp-token-request/)

## Respuesta (`ServerNewOtpTokenResponse`)

| Campo | Tipo | Requerido | Descripción |
|---|---|---|---|
| `sourceFspId` | `string` | Sí | FSP que responde (tu banco). Usa el `destinationFspId` de la solicitud. |
| `destinationFspId` | `string` | Sí | FSP destino de la respuesta. Usa el `sourceFspId` de la solicitud. |
| `partyId` | `string` | Sí | Identificador del Cliente (mismo que el recibido). |
| `onboardRequestId` | `string` | Sí | Identificador del onboarding en curso (mismo que el recibido). |
| `error` | `ErrorResponse` | No | Incluir si no fue posible generar el nuevo OTP. |

Ver referencia completa: [ServerNewOtpTokenResponse](/pch-switch-grpc-clients-doc/reference/server-new-otp-token-response/)

## Cuando el onboarding no existe

Si el `onboardRequestId` no corresponde a ningún onboarding en curso, incluye un `ErrorResponse` en la respuesta.
