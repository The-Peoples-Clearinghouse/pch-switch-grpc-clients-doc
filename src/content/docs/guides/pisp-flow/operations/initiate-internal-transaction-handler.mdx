Cuando un Cliente inicia una transferencia hacia un beneficiario dentro del mismo participante desde una app PISP, el módulo 3ppi del Switch envía a tu banco una solicitud `InitiateInternalTransaction`. A diferencia de una transferencia hacia otro participante, esta operación se resuelve íntegramente dentro del core bancario del participante, sin necesidad de pasar por el Switch.

Para recibir estas solicitudes, debes registrar un **handler** en el cliente gRPC. El cliente invocará esta función cada vez que el módulo 3ppi solicite una transferencia interna a nombre de un Cliente en tu banco.

<Aside type="note">
  Este método es para transferencias **entre cuentas del mismo participante**. Para transferencias hacia otro participante, consulta [Realizar transferencia PISP](/pch-switch-grpc-clients-doc/guides/pisp-flow/operations/initiate-transaction-handler/).
</Aside>

<Aside type="caution">
  Verifica que el `consentToken` sea válido y que la cuenta origen (`accountId`) esté asociada a dicho consentimiento antes de ejecutar el débito. Si el token no es válido, responde con un `ErrorResponse` en lugar de no responder.
</Aside>

## Registrar el handler

<Tabs syncKey="lenguaje">
  <TabItem label="TypeScript">
    ```typescript
    async function initiateInternalTransactionHandler(
      request: ServerInitiateInternalTransactionRequest,
    ): Promise<ServerInitiateInternalTransactionResponse> {
      const partyId          = request.getPartyid();
      const sourceFspId      = request.getSourcefspid();
      const destinationFspId = request.getDestinationfspid();
      const transactionId    = request.getTransactionid();
      const payeeId          = request.getPayeeid();
      const payeeFsp         = request.getPayeefsp();
      const accountId        = request.getAccountid();
      const consentToken     = request.getConsenttoken();
      const currencyCode     = request.getCurrencycode();
      const amount           = request.getAmount();
      const note             = request.getNote();
      const threeppienabled  = request.getThreeppienabled();
      const transactionType  = request.getTransactiontype();

      const response = new ServerInitiateInternalTransactionResponse();
      // La fuente de la respuesta es el banco receptor (el nuestro)
      response.setSourcefspid(destinationFspId);
      // El destino de la respuesta es quien originó la solicitud
      response.setDestinationfspid(sourceFspId);
      response.setPartyid(partyId);
      response.setTransactionid(transactionId);

      // Verifica el consent token y que la cuenta esté autorizada
      if (!consentTokenAutorizaCuenta(consentToken, accountId)) {
        const error = new ErrorResponse();
        error.setCode('UNAUTHORIZED');
        error.setMessage('El consent token no autoriza operaciones sobre la cuenta indicada.');
        response.setError(error);
        return response;
      }

      // Ejecuta la transferencia internamente en el core bancario,
      // sin pasar por el Switch
      const resultado = await ejecutarTransferenciaInterna({
        transactionId,
        payeeId,
        accountId,
        currencyCode,
        amount,
        note,
        transactionType,
      });

      if (!resultado.exitoso) {
        const error = new ErrorResponse();
        error.setCode('TRANSFER_FAILED');
        error.setMessage('No fue posible ejecutar la transferencia interna.');
        response.setError(error);
        return response;
      }

      response.setTransactionstate(resultado.estado);

      return response;
    }

    // Registrar el handler en el cliente gRPC
    grpcClient.setInitiateInternalTransactionRequestHandlerFn(initiateInternalTransactionHandler);
    ```

        $consentToken     = $req->getConsentToken();
        $amount           = $req->getAmount();
        $currencyCode     = $req->getCurrencyCode();
        $note             = $req->getNote();
        // La fuente de la respuesta es el banco receptor (el nuestro)
        $sourceFspId      = $req->getDestinationFspId();
        // El destino de la respuesta es quien originó la solicitud
        $destinationFspId = $req->getSourceFspId();

        // Verifica el consent token y que la cuenta esté autorizada
        if (!consentTokenAutorizaCuenta($consentToken, $accountId)) {
            return (new ServerInitiateInternalTransactionResponse())
                ->setSourceFspId($sourceFspId)
                ->setDestinationFspId($destinationFspId)
                ->setPartyId($partyId)
                ->setTransactionId($transactionId)
                ->setTransactionState('REJECTED')
                ->setError((new ErrorResponse())
                    ->setCode('UNAUTHORIZED')
                    ->setMessage('El consent token no autoriza operaciones sobre la cuenta indicada.'));
        }

        // Ejecuta la transferencia internamente en el core bancario,

            ->setSourceFspId($sourceFspId)
            ->setDestinationFspId($destinationFspId)
            ->setPartyId($partyId)
            ->setTransactionId($transactionId)
            ->setTransactionState('COMPLETED');
    });

## Campos del mensaje entrante (`ServerInitiateInternalTransactionRequest`)

| Campo | Tipo | Descripción |
|---|---|---|
| `partyId` | `string` | Identificador del Cliente pagador (ej. número de celular). |
| `sourceFspId` | `string` | FSP que origina la solicitud (módulo 3ppi del Switch). |
| `destinationFspId` | `string` | FSP al que va dirigida la solicitud (tu banco). |
| `transactionId` | `string` | Identificador único de la transacción generado por el Switch. |
| `payeeId` | `string` | Identificador del beneficiario dentro del mismo participante. |
| `payeeFsp` | `string` | FSP del beneficiario (mismo que el participante). |
| `accountId` | `string` | Identificador de la cuenta origen del Cliente. |
| `consentToken` | `string` | Token de consentimiento del Cliente. |
| `currencyCode` | `string` | Moneda de la transferencia (ej. `'MXN'`). |
| `amount` | `string` | Monto a transferir. |
| `note` | `string` | Concepto o nota de la transferencia. |
| `threeppienabled` | `boolean` | Indica si la transacción fue iniciada mediante PISP. |
| `transactionType` | `string` | Tipo de transacción. |

Ver referencia completa: [ServerInitiateInternalTransactionRequest](/pch-switch-grpc-clients-doc/reference/server-initiate-internal-transaction-request/)

## Respuesta (`ServerInitiateInternalTransactionResponse`)

| Campo | Tipo | Requerido | Descripción |
|---|---|---|---|
| `sourceFspId` | `string` | Sí | FSP que responde (tu banco). Usa el `destinationFspId` de la solicitud. |
| `destinationFspId` | `string` | Sí | FSP destino de la respuesta. Usa el `sourceFspId` de la solicitud. |
| `partyId` | `string` | Sí | Identificador del Cliente (mismo que el recibido). |
| `transactionId` | `string` | Sí | Identificador de la transacción (mismo que el recibido). |
| `transactionState` | `string` | Sí | Estado final de la transacción. |
| `error` | `ErrorResponse` | No | Incluir si no fue posible ejecutar la transferencia. |

Ver referencia completa: [ServerInitiateInternalTransactionResponse](/pch-switch-grpc-clients-doc/reference/server-initiate-internal-transaction-response/)

## Cuando la transferencia falla

Si el `consentToken` no es válido, la cuenta no tiene fondos suficientes, o el beneficiario no existe dentro del participante, incluye un `ErrorResponse` en la respuesta y omite el campo `transactionState`.
