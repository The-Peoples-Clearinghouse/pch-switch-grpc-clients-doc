Cuando un Cliente consulta el historial de transacciones de una cuenta en una app PISP, el módulo 3ppi del Switch envía a tu banco una solicitud `GetAccountTransactions`. Tu banco debe responder con la lista de transacciones de la cuenta indicada, aplicando los filtros recibidos.

Para recibir estas solicitudes, debes registrar un **handler** en el cliente gRPC. El cliente invocará esta función cada vez que el módulo 3ppi solicite el historial de transacciones de un Cliente en tu banco.

<Aside type="caution">
  Verifica que el `consentToken` sea válido y esté asociado a la cuenta solicitada antes de devolver información de transacciones. Si el token no es válido, responde con un `ErrorResponse` en lugar de no responder.
</Aside>

## Registrar el handler

  <TabItem label="TypeScript">
    ```typescript
    async function getAccountTransactionsHandler(
      request: ServerGetAccountTransactionsRequest,
    ): Promise<ServerGetAccountTransactionsResponse> {
      const partyId          = request.getPartyid();
      const sourceFspId      = request.getSourcefspid();
      const destinationFspId = request.getDestinationfspid();
      const accountId        = request.getAccountid();
      const consentToken     = request.getConsenttoken();
      const startDate        = request.getStartdate();
      const endDate          = request.getEnddate();
      const limit            = request.getLimit();
      const categoryFilter   = request.getCategoryfilter();
      const statusFilter     = request.getStatusfilter();
      const transactionId    = request.getTransactionid();

      const response = new ServerGetAccountTransactionsResponse();
      // La fuente de la respuesta es el banco receptor (el nuestro)
      response.setSourcefspid(destinationFspId);
      // El destino de la respuesta es quien originó la solicitud
      response.setDestinationfspid(sourceFspId);
      response.setPartyid(partyId);
      response.setAccountid(accountId);

      // Obtiene las transacciones de la cuenta asociada al consent token,
      // aplicando los filtros recibidos
      const resultado = obtenerTransacciones(consentToken, accountId, {
        startDate,
        endDate,
        limit,
        categoryFilter,
        statusFilter,
        transactionId,
      });

      if (!resultado) {
        const error = new ErrorResponse();
        error.setCode('ACCOUNT_NOT_FOUND');
        error.setMessage('No se encontró la cuenta asociada al consent token proporcionado.');
        response.setError(error);
        return response;
      }

      const transactionsList = resultado.transacciones.map(tx => {
        const transaction = new Transaction();
        transaction.setTransactionid(tx.id);
        transaction.setDate(tx.fecha);
        transaction.setDescription(tx.descripcion);
        transaction.setAmount(tx.monto);
        transaction.setCategory(tx.categoria);
        transaction.setStatus(tx.estado);
        transaction.setPartyid(tx.partyid);
        return transaction;
      });

      const successResponse = new GetAccountTransactionsSuccessResponse();
      successResponse.setTransactionslistList(transactionsList);
      successResponse.setCurrency(resultado.moneda);
      successResponse.setTotalcount(resultado.totalCount);
      response.setSuccessresponse(successResponse);

      return response;
    }

    // Registrar el handler en el cliente gRPC
    grpcClient.setGetAccountTransactionsRequestHandlerFn(getAccountTransactionsHandler);
    ```

    grpcClient.setGetAccountTransactionsRequestHandlerFn(req -> {
        String partyId          = req.getPartyId();
        String accountId        = req.getAccountId();
        String consentToken     = req.getConsentToken();
        String startDate        = req.getStartDate();
        String endDate          = req.getEndDate();
        int    limit            = req.getLimit();
        String categoryFilter   = req.getCategoryFilter();
        String statusFilter     = req.getStatusFilter();
        String transactionId    = req.getTransactionId();
        // La fuente de la respuesta es el banco receptor (el nuestro)
        String sourceFspId      = req.getDestinationFspId();
        // El destino de la respuesta es quien originó la consulta
        String destinationFspId = req.getSourceFspId();

        ServerGetAccountTransactionsResponse.Builder response =
                ServerGetAccountTransactionsResponse.newBuilder()
                        .setSourceFspId(sourceFspId)
                        .setDestinationFspId(destinationFspId)
                        .setPartyId(partyId)
                        .setAccountId(accountId);

        // Obtiene las transacciones de la cuenta asociada al consent token,
        // aplicando los filtros recibidos
        ResultadoTransacciones resultado = obtenerTransacciones(
                consentToken, accountId, startDate, endDate, limit,
                categoryFilter, statusFilter, transactionId);

        if (resultado == null) {
            return response
                    .setError(ErrorResponse.newBuilder()
                            .setCode("ACCOUNT_NOT_FOUND")
                            .setMessage("No se encontró la cuenta asociada al consent token proporcionado.")
                            .build())
                    .build();
        }

        GetAccountTransactionsSuccessResponse.Builder successResponse =
                GetAccountTransactionsSuccessResponse.newBuilder()
                        .setCurrency(resultado.getMoneda())
                        .setTotalCount(resultado.getTotalCount());

        for (Transaccion tx : resultado.getTransacciones()) {
            successResponse.addTransactionsList(Transaction.newBuilder()
                    .setTransactionId(tx.getId())
                    .setDate(tx.getFecha())
                    .setDescription(tx.getDescripcion())
                    .setAmount(tx.getMonto())
                    .setCategory(tx.getCategoria())
                    .setStatus(tx.getEstado())
                    .setPartyId(tx.getPartyId())
                    .build());
        }

        return response
                .setSuccessResponse(successResponse.build())
                .build();
    });

## Campos del mensaje entrante (`ServerGetAccountTransactionsRequest`)

| Campo | Tipo | Descripción |
|---|---|---|
| `partyId` | `string` | Identificador del Cliente (ej. número de celular). |
| `sourceFspId` | `string` | FSP que origina la solicitud (módulo 3ppi del Switch). |
| `destinationFspId` | `string` | FSP al que va dirigida la solicitud (tu banco). |
| `accountId` | `string` | Identificador de la cuenta cuyo historial se solicita. |
| `consentToken` | `string` | Token de consentimiento del Cliente. |
| `startDate` | `string` | Fecha de inicio del rango a consultar (formato `YYYY-MM-DD`). |
| `endDate` | `string` | Fecha de fin del rango a consultar (formato `YYYY-MM-DD`). |
| `limit` | `number` | Número máximo de transacciones a devolver. |
| `categoryFilter` | `string` | Filtra las transacciones por categoría. |
| `statusFilter` | `string` | Filtra las transacciones por estado. |
| `transactionId` | `string` | Si se especifica, devuelve únicamente la transacción con ese identificador. |

Ver referencia completa: [ServerGetAccountTransactionsRequest](/pch-switch-grpc-clients-doc/reference/server-get-account-transactions-request/)

## Respuesta (`ServerGetAccountTransactionsResponse`)

La respuesta puede contener una respuesta exitosa (`successResponse`) o un error (`error`), nunca ambos.

**Campos de nivel superior**

| Campo | Tipo | Requerido | Descripción |
|---|---|---|---|
| `sourceFspId` | `string` | Sí | FSP que responde (tu banco). Usa el `destinationFspId` de la solicitud. |
| `destinationFspId` | `string` | Sí | FSP destino de la respuesta. Usa el `sourceFspId` de la solicitud. |
| `partyId` | `string` | Sí | Identificador del Cliente (mismo que el recibido). |
| `accountId` | `string` | Sí | Identificador de la cuenta (mismo que el recibido). |
| `successResponse` | `GetAccountTransactionsSuccessResponse` | No | Datos del historial. Incluir si la consulta fue exitosa. |
| `error` | `ErrorResponse` | No | Incluir si el consent token no es válido o la cuenta no existe. |

**GetAccountTransactionsSuccessResponse**

| Campo | Tipo | Requerido | Descripción |
|---|---|---|---|
| `transactionsListList` | `Transaction[]` | Sí | Lista de transacciones. |
| `currency` | `string` | Sí | Moneda de la cuenta (ej. `'MXN'`). |
| `totalCount` | `number` | Sí | Total de transacciones que coinciden con los filtros, independientemente del `limit`. |

**Transaction**

| Campo | Tipo | Requerido | Descripción |
|---|---|---|---|
| `transactionId` | `string` | Sí | Identificador único de la transacción. |
| `date` | `string` | Sí | Fecha de la transacción (formato `YYYY-MM-DD`). |
| `description` | `string` | Sí | Descripción o concepto de la transacción. |
| `amount` | `string` | Sí | Monto de la transacción. |
| `category` | `string` | Sí | Categoría de la transacción. |
| `status` | `string` | Sí | Estado de la transacción. |
| `partyId` | `string` | Sí | Identificador de la contraparte de la transacción. |

Ver referencia completa: [ServerGetAccountTransactionsResponse](/pch-switch-grpc-clients-doc/reference/server-get-account-transactions-response/)

## Cuando la cuenta no existe

Si el `consentToken` no corresponde a ningún consentimiento activo o no está asociado a la cuenta indicada, incluye un `ErrorResponse` en la respuesta y omite el campo `successResponse`.
