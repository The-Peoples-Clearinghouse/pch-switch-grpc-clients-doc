El módulo 3ppi del Switch consulta al banco del Cliente las cuentas disponibles mediante `GetCustomerAccounts`. Para recibir estas solicitudes, debes registrar un **handler** en el cliente gRPC. El cliente invocará esta función cada vez que el módulo 3ppi consulte las cuentas de un Cliente en tu banco.

<Aside type="note">
  Este método es utilizado en dos contextos distintos:
  - **Onboarding:** el módulo 3ppi solicita todas las cuentas disponibles del Cliente para que este seleccione cuáles desea vincular a la app PISP. En este caso, el `consentToken` no está presente.
  - **Operaciones normales:** el módulo 3ppi consulta las cuentas ya vinculadas al Cliente. En este caso, el `consentToken` es obligatorio y debe usarse para identificar qué cuentas están asociadas al consentimiento del Cliente.
</Aside>

<Aside type="caution">
  El handler debe responder siempre dentro del tiempo límite. Si el `partyId` no corresponde a ningún Cliente o no tiene cuentas disponibles, responde con un `ErrorResponse` en lugar de no responder.
</Aside>

## Registrar el handler

<Tabs syncKey="lenguaje">
  <TabItem label="TypeScript">
    ```typescript
    async function getCustomerAccountsHandler(
      request: ServerGetCustomerAccountsRequest,
    ): Promise<ServerGetCustomerAccountsResponse> {
      const partyId          = request.getPartyid();
      const sourceFspId      = request.getSourcefspid();
      const destinationFspId = request.getDestinationfspid();
      const isOnboarding     = request.getIsonboarding();

      const response = new ServerGetCustomerAccountsResponse();
      // La fuente de la respuesta es el banco receptor (el nuestro)
      response.setSourcefspid(destinationFspId);
      // El destino de la respuesta es quien originó la consulta
      response.setDestinationfspid(sourceFspId);
      response.setPartyid(partyId);

      // Si no es onboarding, el consent token es obligatorio
      if (!isOnboarding && !request.hasConsenttoken()) {
        const error = new ErrorResponse();
        error.setCode('MISSING_CONSENT_TOKEN');
        error.setMessage('Se requiere el consent token para operaciones fuera del onboarding.');
        response.setError(error);
        return response;
      }

      // Obtener cuentas según el tipo de solicitud:
      // - Onboarding: todas las cuentas disponibles del cliente
      // - Operación normal: solo las cuentas asociadas al consent token
      const cuentas = isOnboarding
        ? obtenerCuentasDelCliente(partyId)
        : obtenerCuentasPorConsentToken(request.getConsenttoken());

      if (!cuentas || cuentas.length === 0) {
        const error = new ErrorResponse();
        error.setCode('CUSTOMER_NOT_FOUND');
        error.setMessage('No se encontraron cuentas asociadas al cliente.');
        response.setError(error);
        return response;
      }

      const accountList = cuentas.map(c => {
        const account = new CustomerAccount();
        account.setAccountid(c.id);
        account.setAccountnickname(c.alias);
        account.setCurrency(c.moneda);
        account.setIsmainaccount(c.esPrincipal);
        return account;
      });
      response.setAccountsList(accountList);

      return response;
    }

    // Registrar el handler en el cliente gRPC
    grpcClient.setGetCustomerAccountsHandlerFn(getCustomerAccountsHandler);
    ```

## Campos del mensaje entrante (`ServerGetCustomerAccountsRequest`)

| Campo | Tipo | Descripción |
|---|---|---|
| `partyId` | `string` | Identificador del Cliente (ej. número de celular). |
| `sourceFspId` | `string` | FSP que origina la consulta (módulo 3ppi del Switch). |
| `destinationFspId` | `string` | FSP al que va dirigida la consulta (tu banco). |
| `isOnboarding` | `boolean` | Indica si la consulta es parte del flujo de onboarding inicial. |
| `consentToken` | `string` _(opcional)_ | Token de consentimiento del Cliente. Solo presente en consultas fuera del onboarding. Usa `hasConsenttoken()` para verificar su existencia antes de llamar a `getConsenttoken()`. |

Ver referencia completa: [ServerGetCustomerAccountsRequest](/pch-switch-grpc-clients-doc/reference/server-get-customer-accounts-request/)

## Respuesta (`ServerGetCustomerAccountsResponse`)

La respuesta puede contener una lista de cuentas (`accountsList`) o un error (`error`), nunca ambos.

**Campos de nivel superior**

| Campo | Tipo | Descripción |
|---|---|---|
| `sourceFspId` | `string` | FSP que responde (tu banco). Usa el `destinationFspId` de la solicitud. |
| `destinationFspId` | `string` | FSP destino de la respuesta. Usa el `sourceFspId` de la solicitud. |
| `partyId` | `string` | Identificador del Cliente (mismo que el recibido). |
| `accountsList` | `CustomerAccount[]` | Lista de cuentas del Cliente. Incluir si el Cliente existe y tiene cuentas. |
| `error` | `ErrorResponse` | Incluir si el Cliente no existe o no tiene cuentas disponibles. |

**CustomerAccount**

| Campo | Tipo | Requerido | Descripción |
|---|---|---|---|
| `accountId` | `string` | Sí | Identificador único de la cuenta. |
| `accountNickname` | `string` | Sí | Nombre o alias de la cuenta para mostrar al Cliente. |
| `currency` | `string` | Sí | Moneda de la cuenta (ej. `'MXN'`). |
| `isMainAccount` | `boolean` | Sí | Indica si esta es la cuenta principal del Cliente para operaciones PISP. |

Ver referencia completa: [ServerGetCustomerAccountsResponse](/pch-switch-grpc-clients-doc/reference/server-get-customer-accounts-response/)

## Cuando el cliente no existe

Si el `partyId` no corresponde a ningún Cliente en tu banco, o el Cliente no tiene cuentas disponibles, incluye un `ErrorResponse` en la respuesta y omite el campo `accountsList`.
