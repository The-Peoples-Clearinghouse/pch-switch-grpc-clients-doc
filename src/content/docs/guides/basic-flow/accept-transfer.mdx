Cuando un participante actúa como banco receptor, el Switch le envía un mensaje `ServerAcceptTransferRequest` solicitando que confirme si la cuenta destino puede recibir los fondos.
El participante debe evaluar la cuenta, actualizar el saldo del beneficiario y responder con `ServerAcceptTransferResponse` dentro del tiempo límite.

Este es el paso 9-11 del flujo de transferencia: ocurre después de que el banco emisor envía `ExecuteTransfer` y antes de que la transferencia sea definitiva.

Para recibir estas solicitudes, debes registrar un **handler** (callback) en el cliente gRPC mediante `setAcceptTransferHanderFn`. El cliente invocará esta función cada vez que el Switch envíe una solicitud de aceptación de transferencia.

<Aside type="caution">
  No responder dentro del tiempo límite equivale a rechazar la transferencia. El handler debe responder siempre, incluso cuando la cuenta no pueda recibir los fondos.
</Aside>

## Registrar el handler

<Tabs syncKey="lenguaje">
  <TabItem label="TypeScript">
    ```typescript
    // Función que maneja la solicitud de aceptar transferencia entrante
    async function acceptTransferHandler(
      request: ServerAcceptTransferRequest,
    ): Promise<ServerAcceptTransferResponse> {
      // Obtener los datos de la solicitud
      const from = request.getFrom();
		  const to = request.getTo();
		  const amount = request.getAmount();
		
		  if (!from || !to || !amount) {
			  throw new Error("Missing required fields in AcceptTransferRequest");
		  }

		  const requestInfo = {
			  transferid: request.getTransferid(),
			  hometransactionid: request.getHometransactionid(),
			  threeppienabled: request.getThreeppienabled(),
			  from: {
				  fspid: from.getFspid(),
				  idtype: from.getIdtype(),
				  idvalue: from.getIdvalue()
			  },
			  to: {
				  fspid: to.getFspid(),
				  idtype: to.getIdtype(),
				  idvalue: to.getIdvalue()
			  },
			  amount: {
				  amount: amount.getAmount(),
				  currencycode: amount.getCurrencycode()
			  },
			  note: request.getNote() || ""
		  };

      // Responder la solicitud
      const response = new ServerAcceptTransferResponse();

      response.setTransferid(requestInfo.transferid);
		  response.setHometransactionid(requestInfo.hometransactionid);
		  response.setThreeppienabled(requestInfo.threeppienabled);
      // El destino de la respuesta es el banco que inició la transferencia
		  response.setDestinationfspid(requestInfo.from.fspid);
      // El origen de la respuesta es el banco receptor (el nuestro)
		  response.setSourcefspid(requestInfo.to.fspid);

      // Ejecuta reglas de negocio para determinar si la cuenta puede recibir los fondos
      if (!laCuentaDestinoPuedeRecibirFondos(requestInfo)) {
        const errorResponse = new ErrorResponse();
        errorResponse.setCode("ACCOUNT_NOT_ACTIVE");
        errorResponse.setMessage("La cuenta no está activa o no puede recibir fondos.");
        response.setError(errorResponse);
      } 

      // Actualizar los saldos del beneficiario

      return response;
    }

    // Registra el handler en el cliente gRPC
    grpcClient.setAcceptTransferHanderFn(acceptTransferHandler);
    ```

    public void onAcceptTransfer(AcceptTransferRequest request) {
        boolean canAccept = verificarCuentaDestino(
            request.getPartyId(), request.getAmount());

        AcceptTransferResponse response = AcceptTransferResponse.newBuilder()
            .setTransferId(request.getTransferId())
            .setAccepted(canAccept)

    }

## Campos del mensaje entrante (`ServerAcceptTransferRequest`)

| Campo | Tipo | Descripción |
|---|---|---|
| `transferId` | `string` | Identificador de la transferencia asignado por la Cámara. |
| `homeTransactionId` | `string` | Identificador interno del banco emisor. |
| `from` | `Party` | Datos del ordenante (banco y cuenta origen). |
| `to` | `Party` | Datos del beneficiario (banco y cuenta destino). |
| `amount` | `Amount` | Monto y moneda de la transferencia. |
| `amountType` | `string` | Tipo de monto (ej. `'SEND'`). |
| `transactionType` | `TransactionType` | Categorización del tipo de transacción. |
| `note` | `string` | Concepto del pago. |
| `threeppiEnabled` | `boolean` | Indica si la transferencia usa flujo 3PPI. |

Ver referencia completa: [ServerAcceptTransferRequest](/pch-switch-grpc-clients-doc/reference/server-accept-transfer-request/)

## Respuesta (`ServerAcceptTransferResponse`)

La respuesta indica al Switch si la transferencia fue aceptada. Si no se incluye `error`, la transferencia se considera aceptada. Si se incluye un `error`, la transferencia es rechazada.

| Campo | Tipo | Descripción |
|---|---|---|
| `transferId` | `string` | Mismo `transferId` del mensaje recibido. |
| `homeTransactionId` | `string` | Mismo `homeTransactionId` del mensaje recibido. |
| `sourceFspId` | `string` | FSP que origina la respuesta (el banco receptor, el nuestro). |
| `destinationFspId` | `string` | FSP destino de la respuesta (el banco emisor). |
| `threeppiEnabled` | `boolean` | Mismo valor recibido en la solicitud. |
| `error` | `ErrorResponse` | Omitir para aceptar. Incluir con `code` y `message` para rechazar. |

Ver referencia completa: [ServerAcceptTransferResponse](/pch-switch-grpc-clients-doc/reference/server-accept-transfer-response/)

## Cuando la cuenta no puede recibir fondos

Si la cuenta destino no está activa o no puede recibir el monto solicitado, incluye un `ErrorResponse` en la respuesta con un código descriptivo.

{/* | Código | Descripción |
|---|---|
| `ACCOUNT_NOT_ACTIVE` | La cuenta destino no está activa. |
| `ACCOUNT_NOT_FOUND` | No se encontró una cuenta asociada al `partyId`. |
| `AMOUNT_EXCEEDS_LIMIT` | El monto supera el límite de recepción configurado. | */}
