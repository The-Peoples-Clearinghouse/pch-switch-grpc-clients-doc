`LookupParty` es el primer paso del flujo de transferencia. Permite resolver la información de la cuenta destino a partir del número de celular del beneficiario. El Switch enruta la solicitud al banco receptor, que confirma si la cuenta está activa y devuelve los datos necesarios para completar la transferencia.

Este paso debe completarse antes de solicitar al usuario el monto y concepto, ya que la respuesta contiene los datos del beneficiario que se presentan para confirmación.

## Enviar la solicitud

<Tabs syncKey="lenguaje">
  <TabItem label="TypeScript">
    ```typescript
    const request = new LookupPartyRequest();
    request.setPartyid('10011110002');
    request.setPartyidtype('MSISDN');
    request.setPartysubidortype('');       // opcional
    request.setCurrencycode('MXN');        // opcional
    request.setOwnerfspid('BANCODESTINO'); // opcional

    // Envía la solicitud al Switch
    const grpcResponse = await grpcClient.partyLookup(request);

    // Maneja la respuesta:
    const error = grpcResponse.getError();
    if (error) {
      console.error('Error del Switch:', error.getCode(), error.getMessage());
      return;       // no continuar el flujo
    }

    const response = grpcResponse.getResponse();
    if (!response) {
      console.error('Respuesta vacía');
      return;       // no continuar el flujo
    }

    const result = {
      sourceFspId:      grpcResponse.getSourcefspid(),
      destinationFspId: grpcResponse.getDestinationfspid(),
      partyId:          response.getPartyid(),
      partyIdType:      response.getPartyidtype(),
      partySubIdOrType: response.getPartysubidortype() || undefined,
      currencyCode:     response.getCurrencycode() || undefined,
      firstName:        response.getFirstname() || undefined,
      middleName:       response.getMiddlename() || undefined,
      lastName:         response.getLastname() || undefined,
      dateOfBirth:      response.getDateofbirth() || undefined,
      extensions:       response.getExtensionsList().map(ext => ({
        key: ext.getKey(),
        value: ext.getValue(),
      })),
    };
    ```

## Parámetros de la solicitud

| Campo | Tipo | Requerido | Descripción |
|---|---|---|---|
| `partyId` | `string` | Sí | Número de celular u otro identificador del beneficiario. |
| `partyIdType` | `string` | Sí | Tipo de identificador. Usa `'MSISDN'` para número de celular. |
| `partySubIdOrType` | `string` | No | Sub-identificador o tipo adicional. |
| `currencyCode` | `string` | No | Moneda de la cuenta destino (ej. `'MXN'`). |
| `ownerFspId` | `string` | No | FSP al que pertenece la cuenta destino, si se conoce de antemano. |

Ver referencia completa: [LookupPartyRequest](/pch-switch-grpc-clients-doc/reference/lookup-party-request/)

## Respuesta exitosa

Cuando el banco receptor confirma que la cuenta está activa, el Switch devuelve un `LookupPartyResponse` con los datos del beneficiario en su campo `response`.

```json
{
  "sourceFspId": "MIBANCO",
  "destinationFspId": "BANCOEJEMPLO",
  "partyId": "5512345678",
  "partyIdType": "MSISDN",
  "firstName": "María",
  "middleName": "García",
  "lastName": "López",
  "dateOfBirth": "1990-05-20",
  "currencyCode": "MXN",
  "extensions": []
}
```

| Campo | Fuente | Descripción |
|---|---|---|
| `sourceFspId` | `grpcResponse` | FSP que origina la respuesta (tu banco). |
| `destinationFspId` | `grpcResponse` | FSP del banco receptor del beneficiario. |
| `partyId` | `response` | Identificador de la cuenta destino. |
| `partyIdType` | `response` | Tipo de identificador de la cuenta destino. |
| `partySubIdOrType` | `response` | Sub-identificador, si aplica. |
| `currencyCode` | `response` | Moneda de la cuenta destino. |
| `firstName` | `response` | Primer nombre del titular. |
| `middleName` | `response` | Segundo nombre del titular. |
| `lastName` | `response` | Apellido(s) del titular. |
| `dateOfBirth` | `response` | Fecha de nacimiento del titular. |
| `extensions` | `response` | Lista de pares clave-valor con datos adicionales. |

Guarda `partyId`, `partyIdType` y `destinationFspId` — son requeridos para el mensaje `ExecuteTransfer`.

Ver referencia completa: [LookupPartyResponse](/pch-switch-grpc-clients-doc/reference/lookup-party-response/)

## Manejo de errores

Si el Switch no puede resolver la consulta, `grpcResponse.getError()` devuelve un objeto con `code` y `message`. De lo contrario devuelve `undefined`.

```typescript
const error = grpcResponse.getError();
if (error) {
  console.error(`[${error.getCode()}] ${error.getMessage()}`);
}
```

{/* | Código | Descripción | Acción recomendada |
|---|---|---|
| `PARTY_NOT_FOUND` | El número de celular no corresponde a ninguna cuenta activa. | Informar al usuario y no continuar el flujo. |
| `TIMEOUT` | No se recibió respuesta del banco receptor a tiempo. | Reintentar con un nuevo `requestId`. | */}

<Aside type="caution">
  Si la respuesta contiene un error, **no continúes** hacia `ExecuteTransfer`. Presentar datos de un beneficiario no resuelto puede resultar en una transferencia fallida o enviada a la cuenta incorrecta.
</Aside>
