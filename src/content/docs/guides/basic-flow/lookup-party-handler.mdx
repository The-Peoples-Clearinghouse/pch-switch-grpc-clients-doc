Cuando otro participante inicia un `LookupParty` hacia una cuenta en tu banco, el Switch te envía un mensaje `ServerPartyInfoRequest` para que confirmes si la cuenta existe y devuelvas los datos del titular.

Para recibir estas solicitudes, debes registrar un **handler** (callback) en el cliente gRPC. El cliente invocará esta función cada vez que el Switch consulte información de un beneficiario en tu banco.

<Aside type="caution">
  El handler debe responder siempre dentro del tiempo límite. Si la cuenta no existe o no puede recibir fondos, responde con un `ErrorResponse` en lugar de no responder.
</Aside>

## Registrar el handler

<Tabs syncKey="lenguaje">
  <TabItem label="TypeScript">
    ```typescript
    async function partyLookupHandler(
      request: ServerPartyInfoRequest,
    ): Promise<LookupPartyResponse> {
      const partyId          = request.getPartyid();
      const partyIdType      = request.getPartyidtype();
      const sourceFspId      = request.getSourcefspid();
      const destinationFspId = request.getDestinationfspid();

      const response = new LookupPartyResponse();
      // La fuente de la respuesta es el banco receptor (el nuestro)
      response.setSourcefspid(destinationFspId);
      // El destino de la respuesta es el banco que originó la consulta
      response.setDestinationfspid(sourceFspId);

      // Ejecuta reglas de negocio para verificar si la cuenta existe
      if (!laCuentaExiste(partyId)) {
        const errorResponse = new ErrorResponse();
        errorResponse.setCode('PARTY_NOT_FOUND');
        errorResponse.setMessage('No se encontró una cuenta asociada al identificador.');
        response.setError(errorResponse);
        return response;
      }

      // Construye la respuesta exitosa con los datos del titular
      const partyResp = new LookupPartySuccessResponse();
      partyResp.setPartyid(partyId);
      partyResp.setPartyidtype(partyIdType);
      partyResp.setFirstname('Juan');
      partyResp.setMiddlename('Alberto');
      partyResp.setLastname('Pérez');
      partyResp.setDateofbirth('2001-03-20');
      partyResp.setExtensionsList([]);
      response.setResponse(partyResp);

      return response;
    }

    // Registra el handler en el cliente gRPC
    grpcClient.setPartyLookupHandlerFn(partyLookupHandler);
    ```

## Campos del mensaje entrante (`ServerPartyInfoRequest`)

| Campo | Tipo | Descripción |
|---|---|---|
| `partyId` | `string` | Identificador de la cuenta a consultar (ej. número de celular). |
| `partyIdType` | `string` | Tipo de identificador (ej. `'MSISDN'`). |
| `sourceFspId` | `string` | FSP que origina la consulta (banco emisor). |
| `destinationFspId` | `string` | FSP al que va dirigida la consulta (tu banco). |

Ver referencia completa: [ServerPartyInfoRequest](/pch-switch-grpc-clients-doc/reference/server-party-info-request/)

## Respuesta (`LookupPartyResponse`)

La respuesta puede contener una respuesta exitosa (`response`) o un error (`error`), nunca ambos.

**Campos de nivel superior**

| Campo | Tipo | Requerido | Descripción |
|---|---|---|---|
| `sourceFspId` | `string` | Sí | FSP que responde (tu banco). Usa el `destinationFspId` de la solicitud. |
| `destinationFspId` | `string` | Sí | FSP destino de la respuesta. Usa el `sourceFspId` de la solicitud. |
| `response` | `LookupPartySuccessResponse` | No | Datos del titular. Incluir si la cuenta existe. |
| `error` | `ErrorResponse` | No | Incluir si la cuenta no existe o hay un error. |

**LookupPartySuccessResponse**

| Campo | Tipo | Requerido | Descripción |
|---|---|---|---|
| `partyId` | `string` | Sí | Identificador de la cuenta (mismo que el recibido). |
| `partyIdType` | `string` | Sí | Tipo de identificador (mismo que el recibido). |
| `firstName` | `string` | Sí | Primer nombre del titular. |
| `middleName` | `string` | No | Segundo nombre del titular. |
| `lastName` | `string` | Sí | Apellido(s) del titular. |
| `dateOfBirth` | `string` | No | Fecha de nacimiento (formato `YYYY-MM-DD`). |
| `extensions` | `Extension[]` | No | Lista de pares clave-valor con datos adicionales. |

Ver referencia completa: [LookupPartyResponse](/pch-switch-grpc-clients-doc/reference/lookup-party-response/)

## Cuando la cuenta no existe

Si el `partyId` no corresponde a ninguna cuenta en tu banco, incluye un `ErrorResponse` en la respuesta y omite el campo `response`.
